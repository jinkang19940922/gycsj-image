# 更新日志 (Changelog)

## [v2.2.0] - 2026-03-15

### 🎯 更新背景与目的

本次更新聚焦于提升用户体验和界面交互质量，主要解决了图片预览功能的冗余问题，优化了全屏显示效果，并新增了展厅模式功能，为用户提供更加沉浸式的图片浏览体验。

---

### ✨ 新增功能

#### 1. 展厅模式 (Gallery Showroom Mode)

**功能描述：**
- 新增展厅模式功能，实现图片流动展示效果
- 支持4列图片同时展示，形成动态画廊效果
- 第1、4列向上流动，第2、3列向下流动，形成视觉对比

**使用说明：**
1. 在图片库页面点击工具栏中的"展厅模式"按钮进入
2. 进入后图片将以流动动画形式展示
3. 鼠标移动时显示退出按钮，静止3秒后自动隐藏
4. 按 ESC 键可快速退出展厅模式
5. 点击右上角的"退出展厅"按钮也可退出

**技术实现：**
- 使用 CSS 动画实现图片列的上下流动效果
- 通过 `duplicateColumnsForShowroom()` 函数复制图片列实现无缝滚动
- 使用 `requestAnimationFrame` 和 GPU 加速优化动画性能
- 添加鼠标移动检测实现按钮的自动显示/隐藏

**界面优化：**
- 全屏覆盖显示，无边缘留白
- 图片保持原始比例，避免变形
- 隐藏所有界面元素（导航栏、工具栏等），提供沉浸式体验
- 添加平滑的过渡动画效果

### 🔧 优化调整

#### 1. 图片预览功能重构

**优化内容：**
- 整合点击图片进入预览界面功能与原图预览功能
- 移除冗余的"原图预览"切换功能
- 简化预览界面，提升用户体验

**实现方式：**
- 修改 `updateModalImage()` 函数，直接使用原图 URL
- 移除 `isOriginalMode` 变量及相关逻辑
- 优化预览按钮的点击事件处理

#### 2. 全屏显示优化

**优化内容：**
- 修复全屏模式下图片四周存在黑色边框的问题
- 优化全屏时图片的显示比例和布局

**实现方式：**
- 修改 `toggleFullscreen()` 函数，动态调整容器尺寸
- 移除 `modalImageContainer` 的固定宽高限制
- 确保图片在全屏模式下能够完整展示

---

### 🐛 问题修复

#### 1. 修复原图预览功能报错

**问题描述：**
- 点击原图预览功能时，系统报错：`ReferenceError: isOriginalMode is not defined`
- 无法正常进入原图预览界面

**解决方案：**
- 移除 `updateModalImage()` 函数中对 `isOriginalMode` 变量的依赖
- 直接使用原图 URL 加载图片
- 清理相关冗余代码

#### 2. 修复展厅模式布局问题

**问题描述：**
- 展厅模式下右侧出现空白区域，未能完全覆盖全屏
- 图片显示高度异常，出现拉伸或压缩变形
- 列间距过大，影响视觉美观

**解决方案：**
- 修改 `.gallery-showroom-mode` 样式，设置 `max-width: none` 移除宽度限制
- 调整图片容器使用 `object-fit: contain` 保持原始比例
- 优化列布局，使用 `flex: 1` 实现均匀分布
- 设置合理的列间距（10px），提升视觉效果

### ⚠️ 可能影响用户使用的变更

#### 1. 功能变更
- **原图预览切换功能已移除**：用户不再需要在预览界面切换原图/缩略图，系统默认直接显示原图
- **展厅模式为新增功能**：不影响现有功能的使用

#### 2. 快捷键变更
- **新增 ESC 键退出展厅模式**：在展厅模式下按 ESC 键可直接退出
- **预览界面快捷键保持不变**：上一张/下一张、放大/缩小、全屏等快捷键功能不变

#### 3. 界面调整
- **预览界面简化**：移除了原图/缩略图切换按钮，界面更加简洁
- **展厅模式为全屏模式**：进入后会隐藏所有界面元素，提供沉浸式体验
